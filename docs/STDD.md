# STDD 规格

这里把 STDD 理解为 Spec/Test Driven Development：先明确规格，再用测试锁住行为，最后实现可演进的代码。

## 目标

建设一个不复杂但边界清晰的量化系统，初期可以用简单策略跑通：

- A 股、港股、美股、BTC 使用同一套标的、行情、组合和交易模型。
- 策略输出目标仓位，不直接操作现金或订单。
- 回测引擎负责按市场规则进行再平衡和交易成本计算。
- 数据源、策略和执行引擎相互解耦，方便后续替换成真实行情和实盘执行。

## 目录设计

```text
configs/                  # 标的池、运行参数等配置
data/sample/              # 示例行情
docs/                     # 架构和规格文档
examples/                 # 可读示例
src/auto_invest/
  domain/                 # 企业级核心领域，不依赖外部框架
  ports/                  # 输入/输出端口接口
  adapters/               # CSV、外部 API、数据库等适配器
  strategies/             # 策略实现
  backtest/               # 回测和绩效
  apps/                   # CLI/服务入口
tests/                    # 行为测试
```

## 行为规格

### 市场支持

- 给定 A 股、港股、美股、BTC 标的配置，系统能够加载为统一 `Instrument`。
- 不同市场可以定义不同的交易数量步长、价格 tick、币种和时区。

### 行情接入

- CSV 行情必须转换为 `Bar`。
- 可以按标的、开始时间、结束时间过滤。
- 输出顺序稳定，按时间和标的排序。

### 策略

- 策略只接收 `MarketSnapshot` 和 `Portfolio`，返回 `TargetWeight`。
- 均线交叉策略在快线高于慢线时持有目标仓位，否则空仓。
- 买入持有策略按固定权重持续持有。

### 回测

- 回测从行情数据构建时间序列快照。
- 引擎根据策略目标仓位生成买卖交易。
- 交易需要考虑手续费、滑点和数量步长。
- 输出权益曲线、交易流水和基础绩效。

## 已落地测试

- `tests/test_market_data.py`：CSV 行情和标的配置加载。
- `tests/test_strategy.py`：均线策略信号。
- `tests/test_backtest.py`：多市场回测、数量步长和绩效指标。
